E-bit

E-bit is a game that revolves around building and designing systems to help the player survive. However, what makes E-bit unique is the game's focus on encouraging cooperation and collaboration among players. By working together, players can share resources, knowledge, and expertise to create more efficient and effective systems that benefit everyone. Whether it's through interacting with NPCs to gain new information or working with other players to achieve goals, E-bit is designed to promote a sense of community and shared success. With rewards for players who collaborate and access to exclusive resources that can only be obtained through cooperative play.

I embarked on developing a game using Unity, leveraging my prior experience with this game engine. During the planning phase, I set specific design goals, including a complete separation of data and game logic from Unity. By utilizing Unity solely as a rendering front-end, the core game mechanics could run independently of Unity's functions, enabling me to build a server-client architecture from scratch. This approach ensures that any dedicated server application created in the future will remain independent of Unity. Additionally, the game was designed with support for mods in mind, and I made sure that entity/object functionality and definitions could be loaded from external files to facilitate this feature.

While my school commitments have limited the time I can devote to game development, I am proactively addressing the issue by redefining the scope of the project. As part of this process, I am researching and defining the game's features and functionality more precisely to ensure that the goals are well-defined from the outset.

One of the key features I am eager to implement is an Entity Component System (ECS) for the game's core. This approach provides a clean and well-organized code structure, along with performance optimizations. By utilizing this architecture, I aim to streamline the game's development process and deliver a more polished final product.

Minetest Mods

Minetest is "An open source voxel game engine" that allowed me to create custom mods for the game. I created several mods that added new seasons, custom blocks, heat, and blight. Additionally, I created a mod that simulated heat moving through the air.

While working on mods for my family's server, I learned how to host and manage changes. This involved several steps before changes could be made. First, the update needed to be extensively tested on a local version. After testing, a meeting would be held with all of the players to discuss the changes and ensure that stakeholders (players) were happy with them. After that, the server would back up the update and apply it before restarting. At the same time, documentation would be updated to reflect the changes.
